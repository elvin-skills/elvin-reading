# 阅读障碍分类与解释协议

一次问题只选最主要的障碍类型，必要时在回答中指出次要障碍。分类的目的不是贴标签，而是避免用“翻译单词”处理所有困难。

## LEX：词汇与专业术语

判定：去掉或误解某个词／短语后，句子的领域含义就错位。

回答顺序：

1. 给出它在当前句中的最短中文义。
2. 指出它与哪些词搭配、承担什么语义角色。
3. 若有旧记忆，并列旧句与新句，比较核心义是否相同。
4. 给一个能跨语境记住的线索；不要罗列整本词典的所有义项。

检索键优先使用词元或领域通用术语，如 `closure`、`opportunity cost`，不用整句中文翻译。

## GRM：语法结构

判定：单词大致认识，但不知道谁修饰谁、动作关系、条件范围或逻辑主干。

回答顺序：

1. 标出句子主干。
2. 用括号或分层说明附加成分。
3. 命名真正可迁移的机制，如 `非限制性定语从句`、`with + 宾语 + 补语`。
4. 与旧例对照“表面不同、机制相同”的部分。
5. 给出下次识别该结构的视觉信号，而不是只翻译本句。

检索键使用稳定的结构名，避免直接使用整条原句。

## BKG：背景知识

判定：句法和词义都懂，但作者省略了该领域默认前提。

回答顺序：

1. 明确指出文本假定读者知道什么。
2. 只补足理解当前段落所需的最小背景。
3. 区分原文事实、通用背景与 Agent 推断。
4. 无法从材料或可靠来源确认时标记 `partial`，不要把猜测写成知识。

## INF：推理障碍

判定：需要把多个句子、前提与结论连接起来，原文并未直接说出答案。

回答顺序：

1. 列出明确证据。
2. 写出从证据到结论的中间桥梁。
3. 标明推断强度：必然、很可能、可能。
4. 给出什么新证据会支持或推翻该推断。

## QST：真实问题

QST 保存的是使用者真正想持续追踪的问题，不是所有“这个词什么意思”。适合 QST 的问题通常满足至少一项：

- 当前材料没有完全解决；
- 后续同领域材料可能提供新证据；
- 会改变使用者对领域的判断或行动；
- 已多次以不同措辞出现。

记录时保存原始问法，并用稳定 `key` 概括问题。新材料解决它时追加 `resolved` occurrence 并链接旧事件。

## CON / OPI / SOL / CAS

- `CON`：多个段落反复依赖的概念，记录定义、边界和代表证据。
- `OPI`：有主体和证据边界的主张；区分作者意见与使用者意见。
- `SOL`：解决什么问题、在什么条件下、通过什么机制生效。
- `CAS`：案例说明什么、不能证明什么，以及它连接哪个概念或主张。

不要把每一章机械总结成五类。没有可靠证据时宁可不记。

## 难以理解时的降阶顺序

先记录使用者的真实反馈，再读取 `adaptive-reading.md` 选择新策略。基本顺序为：

1. 给当前语境下的最短含义。
2. 只保留句子主干。
3. 按意群拆分并重新组合。
4. 替换指代、省略和抽象名词。
5. 使用熟悉的同领域例子，再映射回原句。
6. 拆成更小的 LEX、GRM、BKG 或 INF，并把当前答案标记为 `partial`。

每次解释失败都要更换表示方式。不要用更多术语重复上一轮解释。
